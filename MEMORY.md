# Project Plan: AnyType (Notion-like SaaS)

## Overview

Разработка SaaS-приложения для управления знаниями (аналог Notion / Obsidian):

- Markdown-редактор (block-based, как Notion)
- Работа с медиа (изображения, аудио, PDF и др.)
- Документы как md-страницы
- Авторизация (Yandex + email/password)
- Публичный доступ к документам
- AI-функции (RAG + prompt processing)
- Поиск по контенту
- Темы: light / dark

---

## Tech Stack

- **Frontend**: Next.js 16 + React 19 + MUI + tRPC
- **Backend (AI)**: Python (REST)
- **DB**: PostgreSQL + Prisma
- **Vector DB**: Weaviate
- **Storage**: S3 (MinIO)
- **Cache**: Redis
- **LLM**: Ollama (gemma4)
- **Monorepo**: Turborepo

# ПРАВИЛА ПРЕДМЕТНОЙ ОБЛАСТИ

1. Базовые элементы людей

- большой квадрат — прямой кровный родственник мужского пола
- большой круг — прямая кровная родственница женского пола
- маленький квадрат — брат для прямых кровных родственников
- маленький круг — сестра для прямых кровных родственников
- большой круг с кругом внутри — женщина-владелец генограммы
- большой квадрат с квадратом внутри — мужчина-владелец генограммы
- цифра внутри элемента — порядковый номер партнера
- знак вопроса внутри элемента — неизвестный человек
- внутри фигуры может быть крест — ранняя или трагическая смерть

- для БОЛЬШИХ фигур (основные родственники):
  → текст располагается СЛЕВА от фигуры
- для МАЛЕНЬКИХ фигур и крестов:
  → текст располагается СНИЗУ под элементом

Поддерживаемые данные:

  - каждый союз отдельный

- далее горизонтальная sibling-линия
- от нее вниз к каждому ребенку

8. Близнецы и двойняшки

    - нет горизонтальной линии

Добавь правое навигационное меню для редактора, которое работает как outline / table of contents.

1. Right-side navigation / page outline

- Справа от редактора добавь sticky-навигацию.
- Навигация должна показывать заголовки документа:
  - Heading 1
  - Heading 2
  - Heading 3
- Заголовки должны быть вложены визуально:
  - H1 — основной уровень
  - H2 — с отступом
  - H3 — с большим отступом
- Для каждого пункта показывай текст заголовка.
- Если заголовок пустой, показывай fallback вроде `Untitled heading` или не показывай его, выбрав более аккуратный UX.

2. Поведение при клике

- При клике на пункт навигации редактор должен прокручивать страницу к соответствующему заголовку.
- Скролл должен быть плавным.
- После клика можно установить фокус в редактор и поставить курсор в начало выбранного заголовка, если это не ломает UX.
- Переход должен работать даже при одинаковых текстах заголовков.

3. Active section

- Навигация должна подсвечивать текущий активный заголовок.
- Активный заголовок определяется по текущей позиции скролла.
- Подсветка должна быть мягкой и минималистичной.
- При прокрутке документа active item должен обновляться.
- Если активный заголовок определить сложно, реализуй надежную базовую версию через scroll listener или IntersectionObserver.

4. Dynamic updates

- Меню должно автоматически обновляться, когда пользователь:
  - добавляет новый heading;
  - удаляет heading;
  - меняет текст heading;
  - меняет уровень heading.
- Используй состояние редактора Tiptap и обновляй outline на `editor.on('update')` или через подходящий React/Vue lifecycle.
- Избегай лишних re-render’ов.

5. Empty state

- Если в документе нет заголовков, справа должен быть аккуратный empty state:
  - `Добавьте заголовки, чтобы увидеть навигацию`
- Empty state должен быть ненавязчивым.

6. Responsive behavior

- На больших экранах меню показывается справа от документа.
- На средних экранах можно сделать меню уже или скрываемым.
- На мобильных экранах меню должно скрываться, чтобы не мешать редактору.
- Не ломай layout редактора.
- Основная колонка редактора должна остаться удобной для чтения, примерно `max-width: 720–900px`.

7. Визуальный стиль
   Сделай меню в стиле Notion-like:

- минималистичный дизайн;
- мелкий аккуратный заголовок меню, например `On this page` или `Содержание`;
- нейтральные цвета;
- мягкий hover;
- скругления;
- активный пункт выделяется фоном или левой границей;
- меню не должно отвлекать от текста;
- sticky position с отступом сверху;
- длинные заголовки обрезаются через ellipsis или переносятся аккуратно.

8. Accessibility

- Пункты навигации должны быть кнопками или ссылками с понятным `aria-label`.
- Навигационный блок должен иметь `aria-label`, например `Page outline`.
- Управление мышкой должно быть удобным.
- Hover и active state должны быть визуально различимы.
